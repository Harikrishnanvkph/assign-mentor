ENDPOINTS:

/ - To Show API Guidlines [http request type : GET]

/students - To List Students [http request type : GET]

/mentors - To List Mentors [http request type : GET]

/create/mentor - To Create a Mentor (QUESTION 1)[http request type : POST]

/create/student - To Create a Student (QUESTION 2) [http request type : POST]

/assign/studentMentor - To Assign Student to Mentor (QUESTION : 3 and 4) [http request type : PUT]

/show/mentorStudents/:id - To Show all students for a particular mentor (QUESTION 5) [http request type : GET]

/show/previousMentor/:id - To Show the previously assigned mentor for a particular student (QUESTION :6) [http request type : GET]

/reset - To reset Collection [http request type : DELETE]

To PUT data for Assigning Student to Mentor and vice versa (QUESTION : 3 and 4)

Request body should be of format: { role : "", name : "", assignee : "" } role - should be only "student" or "mentor"
if role is "student" | name : should be student name | assignee - should be mentor name assignee can be null but cannot be array
if role is "mentor" | name : should be mentor name | assignee - should be student name. assignee can be null or array of Student names
examples for student
{
"role" : "student",
"name" : "Rajiv Menon",
"assignee" : "Dr. C. Bhattacharya" OR null
}
examples for mentor
{
"role" : "mentor",
"name" : "Dr. C. Bhattacharya",
"assignee" : "Rajiv Menon" OR null OR ["Rajiv Menon","Rohan Gupta","Pooja Verma"]
}
